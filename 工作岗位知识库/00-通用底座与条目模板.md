# 通用评价底座 · 条目模板

## 一、全行业共通维度（各岗位按需引用，一般不当作「特殊指标」罗列）

以下内容适用于绝大多数实习岗位，可作为导师反馈的底色；写进某一岗位的「评价表」时用一句话引用即可：**「共通项见通用底座」**。

| 维度 | 观察要点（示例，需落到行为/产出） |
|------|--------------------------------------|
| **学习与适应** | 是否在约定周期内（如 2～4 周）读懂团队 wiki/规范；是否输出**可检索**的笔记（目录、链接、待问清单）；第二次同类任务是否减少重复提问 |
| **沟通与协作** | 例会是否带**一页进展**（完成/阻塞/需要决策）；风险是否提前 24～48h 抛出；跨团队是否抄对责任人、附上下文 |
| **执行与闭环** | 任务是否有**可验收定义**（谁、何时、怎样算 done）；承诺日期与实付是否可对照；延期是否同步原因与新时间点 |
| **职业规范** | 密钥/客户数据是否只进密管或脱敏环境；是否走审批拿权限；离开工位是否锁屏 |
| **主动性** | 是否在交付之外留痕**可复用物**（脚本、checklist、FAQ）；改进是否可度量（省多少步、少多少工单） |

**反面信号（便于写鉴定时对照）**：只说「很努力」但无清单/无时间点；重复同类错误且不更新 checklist；把「不知道」停在口头而不落地成文档问题或 mentor 待办。

## 二、单岗位条目模板（复制使用）

请将下列块粘贴到对应分类文件中，替换 `{岗位名称}` 等占位符。

```markdown
### {岗位名称}

- **典型产出**：{用 1～2 句话描述可验收物，如 MR、周报、工单、原型、账本等}
- **特殊评价指标**（相对其他岗位差异化的观察点）
  - **指标 A**：{可观测行为或产出，如能独立排查某类问题、某类图表质量稳定等}
  - **指标 B**：{……}
  - **指标 C**：{……}
- **常见证据**：{Git 提交、工单关闭记录、评审意见、仪表盘、会议纪要、抽样凭证等}

**等级提示（可选用）**：初阶 / 中段 / 高阶可由团队自行定义是否与「独立完成复杂度」「影响范围」挂钩。
```

## 三、「特殊指标」书写原则

1. **可观察**：避免纯形容词（「很强」），改为可核对现象；尽量含 **数量/范围/频率** 之一（如「独立处理 N 个模块」「连续 M 次发布零回滚」需在事实成立时由用户补数）。  
2. **可区分**：若换到相邻岗位仍成立，则归入通用底座，不写入该岗位特殊项。  
3. **与业务对齐**：同一头衔在不同公司差异大，指标最后应能映射到**当前团队 JD 或 OKR**；可显式写「本团队权重高：…」。  
4. **分层写**：同一小节内可用「**及格线** / **加分项**」两段，避免一整段只有空话。  
5. **证据链**：每条指标尽量对应一种**可脱敏留档**的证明方式（工单号、MR 链接、报表截图、底稿页码等）。
