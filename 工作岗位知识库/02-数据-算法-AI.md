> 共通维度见 [00-通用底座与条目模板.md](./00-通用底座与条目模板.md)。

### 算法工程师（搜广推/NLP/CV 等方向）/ 实习

- **典型产出**：实验方案（假设、数据、对照、指标）；训练/推理脚本与**随机种子**记录；离线评测表与**上线/AB** 结论一页纸；特征/模型变更说明。
- **特殊评价指标**
  - **问题形式化（及格线）**
    - 业务目标是否映射到**可优化指标**（如 LogLoss、AUC、NDCG、mAP、延迟分位、业务次留/GMV proxy）并写明**优化的是哪一层**（召回/粗排/精排）。
    - 约束是否列出：QPS、GPU 显存、样本量下限、公平性/合规边界（若有）。
  - **实验规范**
    - 划分：train/val/test 或时间切分是否**防泄漏**（用户、设备、内容侧 leakage 是否检查清单化）。
    - 对照：baseline 是否与**同期数据**可比；是否记录数据版本与特征版本（git hash / 表分区）。
    - 复现：是否固定种子、保存 config；他人是否可按 README **重跑主结果**（误差可接受）。
  - **成本与工程**
    - 训练时长、GPU 卡时、存储是否估算；是否有「精度-成本」折中记录。
    - 推理：batch、量化、蒸馏是否评估过；P99 延迟是否满足业务线要求。
  - **线上与稳健性**
    - 离线涨是否给出**效应量**与置信区间/方差说明（至少不夸大）。
    - 是否讨论**分布漂移**、冷启动、极端流量下的降级（规则/默认排序）。
- **常见证据**：实验 notebook + 导出表、wandb/tensorboard 截图、AB 平台结果（脱敏）、代码 MR 与特征登记工单。

### 机器学习工程 / ML 平台 / 大模型应用 / 实习

- **典型产出**：训练或推理流水线代码；模型/数据 **artifact 清单**（路径、版本、校验和）；评测协议文档；Prompt/RAG 管道配置与**评测集**；Runbook（失败怎么办）。
- **特殊评价指标**
  - **管道可靠**
    - 失败：重试、超时、部分成功是否定义；是否区分**可重跑**与需人工介入。
    - 数据：版本与血缘是否可追溯；是否禁止「读最新无分区」式的隐式漂移。
    - 产出：模型权重、tokenizer、配置是否**捆绑版本**（model card / manifest）。
  - **评测可信度**
    - 离线集：**构造规则**、标签来源、更新频率是否文档化；是否有人类评估抽样协议（双人、rubrics）。
    - LLM：温度、top_p、system prompt 版本是否与报告**同步锁定**。
  - **安全与合规**
    - 训练数据与客户数据是否物理/逻辑隔离；API key 是否来自密管；日志是否脱敏。
  - **可维护性**
    - 新人是否在 1～2 天内能跑一次 **dry-run**；是否有一键命令/Makefile；依赖是否锁版本。
- **常见证据**：CI 任务日志、告警与值班处理单、回滚演练记录、成本看板截图。

### 数据开发 / 数据工程师 / 实习

- **典型产出**：DAG/SQL/作业 YAML；**数据契约**（字段语义、粒度、主键、更新频率）；SLA 与告警规则；血缘截图或文档；回补/重跑 Runbook。
- **特殊评价指标**
  - **数据契约（及格线）**
    - 上下游是否**书面确认**字段含义、分区键、时区、去重规则；breaking change 是否有公告窗口。
  - **幂等、迟到数据与回补**
    - 同一业务日/分区重复跑是否**不翻倍**；watermark 与 late event 策略是否写明。
    - 全量/增量切换是否有过渡方案与验证 SQL。
  - **质量网关**
    - 关键表：非空率、唯一性、及时性（数据要在 T+x 前就绪）是否有断言；告警是否**带负责人与修复链接**。
    - DQ 失败是否区分：阻塞下游 vs 仅通知。
  - **成本与运维**
    - 大任务是否可解释**shuffle/扫描量**；是否做过 partition 裁剪或合并小文件；队列是否抢资源有记录。
- **常见证据**：Airflow/Dolphin 等运行历史、DQ 工单闭环、成本报表、消费方签收聊天记录。

### 数据分析 / 战略数据（偏业务 BI）/ 实习

若强业务决策且需与财务/预测口径对齐，可对照 [07-人力与会计学.md](./07-人力与会计学.md) 中「财务分析（FP&A）」条目。

- **典型产出**：SQL + 看板定义文档（指标口径卡片）；周/月报**固定结构**（结论→证据→局限→建议）；指标体系变更 PR（文档或 wiki）。
- **特殊评价指标**
  - **指标定义（及格线）**
    - 每个核心指标有**公式、分子分母来源表/事件、过滤条件、时间窗口**；是否与财务/数仓「单一事实源」对齐或写明差异原因。
    - 是否标注**尚未对齐**的指标，避免业务会直接用于考核。
  - **统计与因果谦逊**
    - 相关结论是否写清**混杂因素**、样本量、置信度或分群后是否仍成立。
    - 是否避免把平台差异简单归因单一活动（若无对照或实验）。
  - **行动性**
    - 每条结论是否落到 **owner、可执行动作、可验证时间**；是否区分「观察」与「建议」。
  - **工程习惯**
    - SQL 是否可读、有注释；是否封装成**被其他人 import 的视图或 dbt model**（按团队栈）。
    - 看板：权限分级、刷新说明、指标变更是否留版本记录。
- **常见证据**：Looker/Metabase 等定义导出（脱敏）、会议纪要 action、业务方引用数据的邮件/IM（匿名）。
