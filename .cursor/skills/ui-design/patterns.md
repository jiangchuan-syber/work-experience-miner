# UI 设计 — 常见模式（参考）

仅在需要具体方案时阅读，勿默认全部加载。

## 侧栏会话 + 全宽主区 + 固定底栏（本项目 `app.py`）

```
┌ 侧栏 ─────────────┬─ 主区（全宽）──────────────────────────┐
│ API Key            │  [标题]        [简历导入][导出] 右上   │
│ 对话历史（随侧栏滚）│  对话选用、经历卡、对话气泡             │
│ [高级·调试]        │                                        │
└───────────────────┴────────────────────────────────────────┘
┌──────── fixed：jx_chat_input_bar（视口底，宽=stMain） ─┐
│  st.chat_input                                        │
└───────────────────────────────────────────────────────┘
```

| 容器 key | 内容 | 滚动 |
|----------|------|------|
| （侧栏内对话历史） | 历史会话列表 | **勿**嵌套独立滚动区，用 `stSidebar` 默认滚动 |
| `exp_sticky_header` / `exp_card_stack` | 选用、经历字段 | 随页 |
| `jx_chat_scroll` | `st.chat_message` 列表 | 随页；`padding-bottom` 留白 |
| `jx_chat_input_bar` | 仅 `st.chat_input` | **fixed 视口底**（`_pin_chat_input_bar`） |

**勿**再包一层 `jx_chat_dock` 把消息和输入绑在同一滚动流。

## Streamlit 固定底栏实现要点

1. **DOM 顺序**：`st.columns`（历史+主区）→ `_render_chat_messages()` → `_render_chat_input_bar()`（整页最末）。
2. **CSS**：`block-container` 设 `padding-bottom` ≈ 输入条高 + safe-area；消息区 `jx_chat_scroll` 再加底部留白。
3. **对齐主栏**：`components.html` 脚本读取 `section[data-testid="stMain"]` 的 `getBoundingClientRect()`，设置输入条 `left/width`；`resize` + `ResizeObserver` + 侧栏 `aria-expanded` 变化时重算。
4. **验收**：打开含长 AI 回复的会话，**不滚到底**即可在视口底看到输入框并发送。

## 入口命名（本项目已采用）

| 避免 | 改用 |
|------|------|
| `?` popover | `st.expander("使用说明")` |
| `⋯` popover | `st.popover("管理经历")` → 添加/删除 |
| 卡片内「设为对话」 | 仅置顶「对话选用」（`active_exp_index`，`_n_exp > 1` 时显示） |
| caption 写 top-2/向量 | 人话状态一行；技术细节进使用说明 |

## 侧栏 + 主区 + 底栏（通用）

- 侧栏：**改变会话或文件** 的操作；调试进折叠
- 主区：**当前任务** 的编辑与反馈
- 底栏：**高频输入**；fixed + 主区/消息区 bottom padding

## 字段可见性决策树

```
该字段是否完成主路径所必需？
├─ 是 → 主区默认展示
└─ 否 → 用户是否常在本会话调整？
    ├─ 是 → 侧栏或主区次级区（折叠可接受）
    └─ 否 → 设置页 / 高级展开 / 首次向导
```

## 列表 + 详情

- 列表只显示**决策所需列**（名称、状态、时间）
- 详情再展示长文本与元数据
- 避免列表行内塞 10+ 列

## 空态 / 加载 / 错误

| 状态 | 内容 |
|------|------|
| 空态 | 说明为何空 + 一个主 CTA（如上传、新建） |
| 加载 | 骨架或明确「正在…」，禁重复提交 |
| 错误 | 发生了什么 + 用户可做什么（重试、检查格式） |
